# AI剧制作 SOP

> 适用场景：AI短剧、AI短片、AI MV、概念预告片、活动宣传影像、剧情向视频实验  
> 适用对象：刚接触 AI 影像的创作者、黑客松参赛者、内容团队、宣发团队、个人创作者  
> 核心目标：用一套可复用流程，把剧本、视觉、生成、剪辑和交付连接起来

---

## 0. 这份 SOP 解决什么问题

很多人第一次做 AI 影像，会直接打开一个视频生成平台，然后输入一句：

> 一个男生在雨夜追火车，电影感，高清，8K。

大概率会得到一个看起来还行、但不能真正使用的片段。

问题通常不是模型完全不行，而是流程太跳跃。

AI剧制作不是：

```txt
想法 → 提示词 → 成片
```

更合理的流程是：

```txt
想法
→ 剧本
→ 15秒叙事单元
→ 人物与场景资产
→ 视频生成
→ 筛选与重生成
→ 剪辑
→ 声音设计
→ 画质提升
→ 交付与沉淀
```

这份 SOP 的目的，是让团队成员少一点随机抽卡，多一点可控生产。

## 1. 制作前先确认项目类型

开始前先判断你要做的不是“视频”，而是哪一种视频。

### A. AI短剧

特点：

- 有角色。
- 有剧情。
- 有情绪变化。
- 有对白或旁白。
- 对人物一致性要求较高。

适合流程：

```txt
剧本 → 分场 → 人物设定 → 15秒生成 → 剪辑成片
```

### B. AI短片

特点：

- 更重视氛围、影像感和节奏。
- 可以少对白。
- 可以用视觉隐喻。
- 对画面质感要求较高。

适合流程：

```txt
主题 → 影像段落 → 视觉母版 → 镜头生成 → 声音设计
```

### C. AI MV

特点：

- 音乐先行。
- 情绪和节奏比剧情更重要。
- 可以接受抽象画面。
- 镜头连续性压力较低。

适合流程：

```txt
音乐分析 → 情绪段落 → 视觉风格 → 镜头生成 → 卡点剪辑
```

### D. 活动宣传片 / 概念预告

特点：

- 信息传达明确。
- 风格统一重要。
- 不一定需要复杂剧情。
- 适合社群、公众号、小红书、路演开场。

适合流程：

```txt
核心信息 → 视觉概念 → 关键画面 → 动效生成 → 包装输出
```

## 2. AI影像的两种工作流

### 2.1 1.0 工作流：逐镜头拼装

早期 AI 影像制作通常比较重，需要先准备：

- 人物设定。
- 场景设定。
- 服装道具。
- 视觉母版。
- 多机位画面。
- 每个镜头的提示词。
- 人物一致性方案。
- 场景连续性方案。

优点：

- 可控性更强。
- 适合关键镜头。
- 适合商业展示。
- 适合需要人物稳定的项目。

缺点：

- 制作慢。
- 分镜成本高。
- 资产管理复杂。
- 需要大量测试。

### 2.2 2.0 工作流：15秒叙事单元

新一代视频模型开始更能理解一段戏。可以尝试把剧情拆成多个 15 秒左右的叙事单元，而不是一开始就拆成几十个细碎镜头。

一个 15 秒叙事单元通常包含：

- 一个场景。
- 一个核心人物或人物关系。
- 一个主要动作。
- 一次情绪变化。
- 一个进入点。
- 一个退出点。

示例：

```txt
雨夜站台，男主发现火车即将离开。他攥着一盘旧磁带追赶火车，雨水打湿外套，火车灯光在远处闪烁。镜头从远景逐渐推进，最后停在他崩溃但克制的表情上，整体像低饱和度独立爱情电影。
```

优点：

- 效率高。
- 适合快速试错。
- 更适合新手上手。
- 适合短片、MV、概念片段。

缺点：

- 结果不一定稳定。
- 关键动作可能失控。
- 人物一致性仍然需要处理。
- 复杂剧情不能完全放手。

### 2.3 推荐方式：混合工作流

实际项目中建议使用：

```txt
2.0 快速生成整体段落
+
1.0 精准控制关键镜头
```

即：

- 氛围镜头：让模型自由发挥。
- 普通剧情镜头：使用半控制生成。
- 关键镜头：回到人物资产、场景母版、首帧控制。

## 5. 阶段二：剧本整理

AI短剧最容易失败的原因之一，是一开始写得太复杂。

建议先写：

- 一个核心人物。
- 一个核心场景。
- 一个核心冲突。
- 一个情绪转折。
- 一个视觉记忆点。

剧本最小模板：

```txt
故事一句话：
主角是谁：
他/她想要什么：
阻碍是什么：
情绪如何变化：
最后停在哪个画面：
```

示例：

```txt
故事一句话：
一个隐居的音乐人，在雨夜追赶一列即将离开的火车，试图把一盘旧磁带交给曾经的爱人。

主角是谁：
年轻男音乐人，克制、敏感、孤独。

他想要什么：
把一盘旧磁带交出去。

阻碍是什么：
火车即将离开，对方没有回头。

情绪如何变化：
压抑 → 急切 → 崩溃 → 接受。

最后停在哪个画面：
他站在雨中的站台尽头，低头看着被雨水打湿的磁带。
```

## 6. 阶段三：拆分 15 秒叙事单元

每个 15 秒单元只做一件事。

不要一个单元里同时出现：

- 换场景。
- 多人物争吵。
- 大量对白。
- 复杂动作。
- 闪回。
- 道具交互。
- 情绪大反转。

这会让模型迷路。模型一迷路，预算就开始燃烧。

15秒单元模板：

```txt
单元编号：
时长：
场景：
人物：
核心动作：
情绪变化：
进入点：
退出点：
画面重点：
镜头倾向：
生成方式：
风险点：
```

示例：

```txt
单元编号：S01
时长：15 秒
场景：雨夜火车站台
人物：男主
核心动作：男主追赶即将离开的火车
情绪变化：压抑 → 急切
进入点：男主发现火车开始移动
退出点：男主停在站台尽头
画面重点：雨水、火车灯光、旧磁带、奔跑
镜头倾向：远景推进到中近景
生成方式：文本生成 + 人物参考图
风险点：人物变脸、跑步动作变形、火车结构混乱
```

## 7. 阶段四：人物设定

以下情况建议做人设：

- 主角多次出现。
- 有多个镜头特写。
- 有正脸、侧脸、背影。
- 需要统一服装。
- 需要稳定年龄和气质。
- 后期还要做海报、封面、宣发图。

人物卡模板：

```txt
角色名：
年龄：
身份：
气质关键词：
发型：
五官特征：
身材：
服装：
配饰：
禁止变化：
参考风格：
```

示例：

```txt
角色名：男主
年龄：22-25
身份：隐居音乐人 / 独立音乐制作人
气质关键词：克制、孤独、敏感、冷感、疲惫
发型：黑色中长发，略微凌乱
五官特征：清瘦，眼神疲惫但专注
身材：偏瘦，肩膀略窄
服装：深色旧外套、灰黑色内搭、湿润质感
配饰：旧磁带、银色戒指
禁止变化：不要变成商务男、不要欧美硬汉感、不要过度精致
参考风格：低饱和度独立电影人物
```

人物资产建议至少准备：

```txt
正面图
侧面图
背面图
半身特写
全身图
核心服装版本
情绪表情版本
```

命名建议：

```txt
character_male_main_front_v01.png
character_male_main_side_v01.png
character_male_main_back_v01.png
character_male_main_closeup_v01.png
```

## 8. 阶段五：场景与视觉母版

场景母版用于统一：

- 空间结构。
- 色彩。
- 光线。
- 氛围。
- 时代感。
- 摄影风格。

如果没有场景母版，每个镜头都可能像来自不同剧组。

场景设定模板：

```txt
场景名：
时间：
天气：
空间结构：
主要物件：
光源：
色彩：
氛围：
禁止元素：
```

示例：

```txt
场景名：雨夜站台
时间：深夜
天气：大雨
空间结构：长站台、远处铁轨、旧式顶棚、昏暗站灯
主要物件：火车、站牌、积水、旧磁带
光源：远处火车灯、站台顶灯、雨水反光
色彩：深蓝、黑灰、低饱和暖黄
氛围：孤独、潮湿、克制、电影感
禁止元素：现代高铁站、明亮商业灯光、过度赛博朋克
```

视觉后缀模板：

```txt
低饱和度独立电影质感，克制构图，深色调，真实雨夜光线，浅景深，胶片颗粒，冷暖对比，人物情绪含蓄，镜头语言克制，不要广告感，不要过度美颜，不要赛博朋克霓虹。
```

注意：视觉后缀不能机械复制。白天、夜晚、室内、室外、特写、远景都要适当调整。

## 9. 阶段六：分镜与镜头等级

不要所有镜头都同等控制。把镜头分成三类。

### A类：自由生成镜头

适合：

- 空镜。
- 氛围。
- 环境。
- 情绪过渡。
- 抽象画面。
- 光影变化。

生成方式：

```txt
文本提示词直接生成
```

### B类：半控制镜头

适合：

- 普通人物动作。
- 普通剧情推进。
- 风格统一镜头。
- 不要求精确动作的镜头。

生成方式：

```txt
文本提示词 + 人物参考图 / 场景参考图
```

### C类：强控制镜头

适合：

- 人物特写。
- 核心表演。
- 关键道具。
- 产品展示。
- 复杂动作。
- 结尾画面。
- 海报级画面。

生成方式：

```txt
人物资产 + 场景母版 + 首帧控制 + 明确镜头提示
```

镜头表模板：

```txt
镜头编号：
所属单元：
镜头等级：A / B / C
画面内容：
人物：
动作：
景别：
机位：
运镜：
光线：
情绪：
生成平台：
参考资产：
风险点：
是否重生成：
```

## 10. 阶段七：平台选择

平台不是排行榜。

不要问“哪个模型最强”，应该问：

```txt
这个镜头适合哪个平台？
```

### 即梦 / Seedance

适合：

- 中文创作。
- 快速试错。
- 15秒叙事段落。
- MV片段。
- 情绪镜头。
- 短片实验。

注意：

- 复杂动作仍然可能失控。
- 关键人物镜头建议加参考图。

### Google Omni / Flow

适合：

- 剧本视觉开发。
- 角色设计。
- 场景母版。
- 改图。
- 首帧设计。
- 项目式素材管理。

注意：

- 更适合放在前期视觉开发和项目管理中，不只是单次生成。

### 可灵 / 快乐马 / 其他平台

适合：

- 作为备用生成路径。
- 交叉测试不同题材。
- 对比动作、质感和稳定性。
- 某些镜头死磕失败时换平台。

### LibTV 导演台类工具

适合观察：

- 剧本导入。
- 自动拆镜头。
- 角色场景管理。
- 批量生成。
- 导演工作台式生产。

注意：

- 当前更适合作为研究对象和辅助工具，不要完全依赖。

平台选择表：

```txt
镜头类型：氛围 / 人物 / 动作 / 特写 / 产品 / 空镜
控制需求：低 / 中 / 高
成本预算：低 / 中 / 高
是否需要参考图：是 / 否
首选平台：
备用平台：
失败后策略：
```

## 11. 阶段八：提示词写法

提示词不是堆形容词。

低质量提示词：

```txt
一个男生在雨夜追火车，电影感，高级，高清，8K，史诗感。
```

问题：

- 人物不清楚。
- 场景不清楚。
- 动作不清楚。
- 镜头不清楚。
- 情绪不清楚。
- 模型自由发挥过大。

视频提示词结构：

```txt
主体：
场景：
动作：
情绪：
镜头：
光线：
风格：
时长：
禁止事项：
```

示例：

```txt
主体：
一位年轻男音乐人，黑色中长发，清瘦，穿深色旧外套，手里攥着一盘旧磁带。

场景：
深夜雨中的老式火车站台，远处火车即将离开，站台灯光昏暗，地面有积水反光。

动作：
他从站台中段开始追赶火车，脚步越来越急，最后停在站台尽头，低头喘息。

情绪：
压抑、急切、克制的崩溃，不要夸张哭喊。

镜头：
从远景缓慢推进到中近景，轻微手持感，最后停在他的侧脸和磁带上。

光线：
低饱和夜景，冷色雨夜，远处火车灯带来微弱暖光。

风格：
独立爱情电影，胶片颗粒，克制构图，真实光线。

时长：
约 15 秒。

禁止事项：
不要现代高铁站，不要赛博朋克霓虹，不要商业广告质感，不要人物突然变脸，不要夸张动作。
```

## 12. 阶段九：LLM 辅助流程

LLM 不只是帮你写提示词。

它可以参与：

- 剧本打磨。
- 分镜拆解。
- 镜头风险判断。
- 风格统一。
- 平台提示词改写。
- 失败结果复盘。
- 生成工作流整理。

推荐提示词详见：

```txt
prompts/script-polish.md
prompts/video-prompt.md
```

## 13. 阶段十：生成与筛选

不要一次只生成一个版本。

建议每个重要镜头至少生成：

```txt
A版：自由发挥
B版：加强人物
C版：加强场景
D版：加强镜头
```

然后比较：

- 哪个动作最好？
- 哪个人物最稳？
- 哪个画面最有质感？
- 哪个最适合剪辑？
- 哪个可以作为首帧继续生成？

筛选标准：

```txt
人物是否稳定：
动作是否自然：
画面是否有记忆点：
是否能接上下一个镜头：
是否有明显穿帮：
是否符合整体风格：
是否值得花时间后期修：
```

如果一个镜头动作很好但人物不稳，可以尝试修。  
如果一个镜头从结构上就错了，不要硬救。

硬救坏镜头，通常比重生成更贵。

## 14. 阶段十一：剪辑与声音

AI视频必须剪辑。AI生成片段通常节奏不稳定。

剪辑要做的事：

- 删除废帧。
- 压缩无效动作。
- 调整镜头顺序。
- 用空镜建立节奏。
- 用声音掩盖轻微视觉问题。
- 用字幕和音乐强化情绪。
- 避免连续展示模型缺陷。

声音可以显著提升 AI 视频质感。建议加入：

- 环境声。
- 脚步声。
- 雨声。
- 交通声。
- 房间底噪。
- 呼吸声。
- 音乐。
- 低频氛围。
- 转场音效。

很多 AI 视频看起来廉价，不完全是画面问题，是因为它没有声音空间。

## 15. 阶段十二：画质提升

AI视频生成后常见问题：

- 分辨率低。
- 画面糊。
- 人脸不稳定。
- 闪烁。
- 运动拖影。
- 噪点。
- 压缩感。
- 帧率不足。

### Topaz AI 类工具

适合：

- 视频超分。
- 降噪。
- 插帧。
- 细节恢复。
- 运动修复。

注意：

- 不要参数拉满。
- 过度增强会产生塑料感。
- 人脸和文字最容易被修坏。

### RunningHub / ComfyUI 工作流社区

适合找：

- 超分工作流。
- 插帧工作流。
- 去闪烁工作流。
- 人脸修复工作流。
- 批量处理工作流。
- 风格统一工作流。

工作流社区的价值不是单纯找免费工具，而是学习别人怎么拆解任务。

## 18. 给新手的最小可行路径

如果你是第一次做 AI 剧，不要直接做 3 分钟短片。

建议先做一个 30 秒片段。

```txt
总时长：30 秒
结构：2 个 15 秒单元
人物：1 人
场景：1 个
情绪：1 次变化
对白：无或极少
输出：1080p MP4
```

推荐主题：

```txt
一个人深夜回到空房间。
一个人在雨中等待没有来的人。
一个人在旧电脑前听到一段录音。
一个人打开抽屉，发现一张旧照片。
一个人在天台看城市灯光。
```

最小流程：

```txt
一句话故事
↓
拆成 2 个 15 秒单元
↓
做 1 张人物参考图
↓
做 1 张场景母版
↓
每个单元生成 3 个版本
↓
挑 2 个可用片段
↓
剪辑 + 音乐 + 字幕
↓
输出成片
```

## 19. 最后原则

### 不要迷信单个模型

模型只是链路中的一环。

真正决定结果的是：

```txt
剧本是否适合生成
视觉是否明确
镜头是否合理
提示词是否清楚
素材是否统一
剪辑是否有效
声音是否补足画面
后期是否完成交付
```

### 不要把提示词当魔法

提示词不是咒语。它更像导演说明。

你要告诉模型：

```txt
谁
在哪里
做什么
情绪如何变化
镜头怎么看
光线是什么
不要出现什么
```

### 不要一次追求大片

先做短。先做稳。先做一个能交付的 15 秒。

比做一个崩掉的 3 分钟更有价值。

### 最重要的能力

AI 提高的是执行速度。

但下面这些仍然需要人负责：

```txt
方向
审美
判断
取舍
结构
节奏
最终交付
```

这也是 AI 剧制作真正的门槛。
